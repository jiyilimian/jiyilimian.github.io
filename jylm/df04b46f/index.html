0x02.Chunk机制

Spring batch在配置Step时采用的是基于Chunk的机制。
1.每次读取一条数据，再处理一条数据，累积到一定数量后再一次性交给writer进行写入操作。
2.最大化的优化写入效率，整个事务也是基于Chunk来进行。

注：
1.在需要将数据写入到文件、数据库中之类的操作时可以适当设置Chunk的值以满足写入效率最大化。
2.有些场景下我们的写入操作其实是调用一个web service或者将消息发送到某个消息队列中，那么这些场景下我们就需要设置Chunk的值为1，这样既可以及时的处理写入，也不会由于整个Chunk中发生异常后，在重试时出现重复调用服务或者重复发送消息的情况。

0x03.Listener机制

Spring batch提供了大量的Listener来对job的各个执行环节进行全面的监控。
实现方式：
1.继承自相应的接口，比如继承JobExecutionListener接口等。
2.使用annoation（注解）的方式。使用注解的方式更灵活一些，因为继承接口需要实现接口的所有方法，而使用注解则只需要在相应的方法添加注解。如@OnSkipInRead 读操作跳过操作等。

0x05.并发处理机制

Spring Batch的并发处理机制在处理海量数据是必不可少的，实现方式有以下几种：

  • 在单个step中多线程执行任务：

    上述示例中的task let需要实现Task Executor，Spring Batch提供了一个简单的多线程Task Executor供我们使用：Simple Async Task Executor。
    可以借助于task Executor 来实现。这种情况适合于reader、writer是线程安全且是无状态的场景。我们还可以设置线程数量。

  • 并行执行同一个Step：
    Spring batch提供了Partition Step来实现对同一个step在多个进程中实现并行处理。通过 Partition Step再配合Partition Handler可以将一个step扩展到多个Slave上实现并行运行。

  • 并行执行不同的Step：

    在这个示例中我们先执行step1，然后并行执行flow1和flow2，最后再执行step3

  • 远程执行Chunk任务：
    远程执行Chunk任务则是将某个Step的processer操作分割到多个进程中，多个进程通过一些中间件进行通讯（比如采用消息的方式）。这种方式适合于Processer是瓶颈而Reader和Writer不是瓶颈的场景。
